# Security policy

## Supported versions

Security fixes are provided for the latest released version of WP Swiper. Users should update before reporting an issue that may already be resolved.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's **Report a vulnerability** option on the repository Security page. This creates a private security advisory visible only to the reporter and repository maintainers. If private vulnerability reporting is unavailable, use the [Digital Apps contact page](https://digitalapps.com/contacts/) and clearly identify the report as a WP Swiper security issue.

Please include:

- A clear description of the issue and its potential impact
- The affected plugin and WordPress versions
- Reproduction steps or a minimal proof of concept
- Any suggested mitigation

Remove credentials, personal data, and unrelated production information from the report. We aim to acknowledge complete reports within seven days and will coordinate disclosure after a fix is available.
